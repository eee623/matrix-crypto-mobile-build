# Security policy

Do not report application or user data here because this repository must never
receive such information. Report vulnerabilities in the public build scripts
through GitHub's private vulnerability reporting feature when available.

The accepted public surface is limited to public dependency pins, build-only
automation, generated bindings/native libraries, manifests, checksums, and
license notices. Secrets, endpoints, identifiers, cryptographic stores, keys,
tokens, cookies, private source, caches, and production configuration are out
of scope and must not be committed or uploaded as artifacts.
